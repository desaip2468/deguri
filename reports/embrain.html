엠뇌 경진대회 발표 대본

1p

  • 안녕하세요. 데이터 굴삭기의 줄임말인 데구리 팀입니다. 저희 팀의 발표 주제는 오공육공 세대의 간편 결제 이용률 증가를 위한 방안 제안입니다. 발표는 다음과 같은 순서로 진행하겠습니다.

2p

  • 먼저 문제정의입니다.

3p

  • 외부 리서치와 엠브레인 데이터를 분석한 자료를 보시면 5060 세대는 다른 연령대에 비해 낮은 간편 결제 이용률을 보인다는 사실을 알 수 있습니다. 그 원인을 조사해본 결과 이는 5060세대가 디지털 금융 환경에 익숙하지 않고, 현재 대부분의 간편결제 마케팅전략이 2030세대를 주타겟으로 하고 있기 때문이었습니다. 그러나 ‘베이비붐’세대인 5060세대는 높은 인구 비율과 안정적인 경제력을 바탕으로 간편 결제 시장의 ‘큰 손’이 될 잠재력이 충분합니다.

 

4p

  • 또한 패널자료를 분석해본결과 2030세대는 네이버페이, 카카오페이 등의 뚜렷한 강세를 보이는 간편 결제 회사가 있는 반면, 5060세대는 간편결제 회사들이 비슷한 시장점유율을 보였습니다. 따라서 간편결제회사가 5060세대를 공략할시, 상당한 이윤창출과 함께 시장선점효과까지 누릴 수 있을 것이라 기대하며, 저희는 데이터 분석을 통해 오공육공 세대를 효과적으로 공략하기 위한 방안을 찾아보았습니다.

 

5p

 

6p

  • 다음으로는, 저희가 데이터 분석 전 수행했던 EDA 과정에 대해서 말씀드리도록 하겠습니다. 저희가 main 으로 활용한 데이터는 시간순으로 정렬된 Payment History Data 였습니다. 저희는 사용자별 소비 형태에 대한 insight 를 얻고 싶었기 때문에, History Data 에 존재하는 컬럼별로 나름의 기준을 세워 aggregation 을 진행 하였습니다.
  • 예를 들면, PANEL_ID 가 "A" 인 Payment History Data 가 이와 같이 5건 있을 경우, "Category Code" 컬럼에 대해서는 PANEL 별로 count 값을 사용하자 라고 결정한 후, 각 value 의 빈도수를 세어 이 값을 PANEL 의 Category Code 를 대표하는 값으로 삼았습니다.

 

7p

  • PANEL 별로 Payment History 정보를 aggregate 시킨 후, 다음으로 저희 모델의 타겟 변수와 설명 변수를 선택하는 과정이 있었습니다. 저희의 목적은 5060 세대의 간편 결제 이용을 촉진하는 것이었으므로, 데이터 중 간편결제 이용 상황을 가장 잘 대변하는 값은 PANEL 별 간편결제 이용 승인 금액이라고 생각하여 이를 타겟 변수로 삼았습니다.
  • 이후 저희는 추가적으로 설문지 데이터와 APP 이용시간 데이터에 대해서도 앞서 말씀드린 Payment History Data 와 같은 aggregation process 를 진행하였습니다. 이 과정을 통해 PANEL_ID 별로 묶인 통합 데이터를 생성하였고, 이 데이터에서 간편 결제 이용액에 영향을 미칠 수 있다고 생각되는 모든 다른 컬럼을 설명 변수로 잡았습니다.

 

8p

  • 다음은 5060 세대 패널들의 간편결제 서비스 이용 여부와, 이용액을 시각화한 자료를 보시겠습니다. 왼쪽 표는 5-60대의 온라인/오프라인별 간편결제 서비스 이용률을 표로 나타낸 것이고, 오른쪽 그림은 5060 패널의 간편결제 승인 금액을 역순으로 정렬한 그래프입니다. 왼쪽 표를 보시고 알 수 있는 사실은, 5,60대의 경우에는 간편 결제 이용시 현금이나 신용카드를 사용하는 비율이 압도적으로 높다는 것입니다. 오른쪽 그림을 보고 알 수 있는 사실은, 0이 압도적으로 많으므로 전반적인 사용률이 적다는 것과, 일부의 Heavy-user 와 다수의 Light-user 로 구성되어 있다는 것입니다.

 

9p

  • 다음 표는 5-60대 간편결제 이용 / 미이용자의 비율을 나타내고 있습니다. 간편 결제 승인 금액으로 구분한 결과, 간편 결제 이용자 수는 미이용자 수의 절반에도 미치지 못하는 것을 알 수 있었습니다. 간편결제 이용자 수가 적어 이후 구축한 모델에서의 민감도가 낮아지는 문제가 발생하였기 때문에, 이 시점 이후 분석부터는 SMOTE 를 사용하여 간편결제 이용자와 미이용자의 비율을 비슷하게 맞춘 데이터를 사용하여 진행하였습니다.

 

10p

  • 마지막으로 일부 설명 변수들에 대한 추가적인 전처리를 한 내용을 말씀 드리겠습니다. 처음에는 데이터에 주어진 각종 카테고리 구분 사항을 그대로 따라 진행하려고 했으나, 너무 잘게 쪼개져 있어서 데이터 수가 적고, 관련있는 카테고리끼리 묶이지 않아 다중공선성 문제가 심각하게 발생하였습니다. 따라서 몇몇 설명 변수에 대해서는 임의의 기준을 추가로 세워 이 기준에 맞춘 2차 aggregation 을 진행하였습니다. 예로 나와 있는 테이블의 경우, 차량과 관련된 값이 담겨 있는 컬럼들의 값을 모두 하나로 합쳐 "차량" 컬럼으로 대표시킨 것을 보여주고 있습니다.

 

11p

  • 이후에는 제가 이어서, 저희가 사용했던 데이터 분석 방법에 대하여 말씀 드리겠습니다.

 

12p

  • 저희가 가장 먼저 한 것은, 사용자별 간편결제 승인 금액을 바탕으로 군집을 나누는 것이었습니다. Boxplot 을 그려서 알 수 있었던 사실은, 승인 금액의 분포가 굉장히 Right-skewed 되어 있다는 것이었습니다. 이런 경우에 평균은 집단의 특성을 제대로 대표해주지 못할 가능성이 높기 때문에, K-means 대신 중앙값을 사용하는 K-medoids clustering 기법을 사용하였습니다.

 

13p

  • K-medoids clustering 기법을 사용하였을 때 저희 5060 패널 데이터는 3개의 cluster 로 나뉘었습니다. 이는 EDA 과정에서 관찰된 내용과도 일치하는 결과였습니다. 이후 저희는 분류된 내용을 바탕으로 군집별 특성을 파악하고, 군집별 차이가 유의한지 다양한 검정을 시도해 보았습니다.

 

14p

  • 저희 데이터를 설명할 수 있는 적절한 모델을 고르기 위해 SAS E-miner 의 모델 비교 기능을 사용해 보았습니다. 이 결과, 로지스틱 회귀 모델이 오차가 가장 적게 저희 데이터를 설명해 주는 것으로 나타났습니다.

 

15p

  • 저희는 따라서 몇 가지 데이터에 대하여 로지스틱 회귀 분석을 수행해 보았습니다. 먼저 저희는 가맹점 카테고리 코드를 설명 변수를 놓고 간편 결제 사용 여부를 타겟 변수로 놓는 로지스틱 회귀 분석을 실행하였습니다. 모델에 대해 test data 로 validation 을 수행한 결과 72% 정도의 accruacy 를 가지는 준수한 모델이 만들어졌습니다.

 

16p

  • 다음으로 패널별 app 사용 시간에 대해서도 같은 방법으로 로지스틱 회귀 분석을 하였고, 74% 정도의 accuracy 를 가지는 모델을 만들어 낼 수 있었습니다.
